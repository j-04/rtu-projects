Description of the task:
The main process provides input / output of information and control of child flows. In total, it creates 5 threads. At the command of the user, these threads are simultaneously started, they wait for a random time interval, after which they stop and report a stop to the main thread. The main process displays a message about each thread on the screen. You cannot terminate threads. When all threads finish / finish work, the user will be prompted with the main process to retry / redo or exit. When you repeat, it starts from the beginning, when you exit, each thread waits for a random time, and then completes its work. The main process reports the completion of each thread on the screen, after which it finishes its work. The main process cannot kill child threads. Threads should not be restarted.
